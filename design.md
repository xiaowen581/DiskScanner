# DiskScanner — 磁盘空间分析工具 软件设计书

---

## 1. 项目概述

### 1.1 项目名称

**DiskScanner** — 磁盘文件/文件夹空间分析与清理辅助工具

### 1.2 背景与目标

随着日常使用中文件不断积累，磁盘空间逐渐被占满，用户往往难以快速定位哪些文件或文件夹占用了最多空间。本工具旨在提供一个**交互式终端界面（TUI）**，帮助用户：

- 快速扫描指定目录及其所有子目录
- 按文件大小从大到小排序展示
- 同时统计各文件夹的总大小
- 支持交互式浏览，方便用户定位并决定是否删除

### 1.3 技术选型

| 项目       | 选型                    | 理由                                              |
| ---------- | ----------------------- | ------------------------------------------------- |
| 语言       | **Python 3.8+**         | 跨平台、标准库丰富、开发效率高                     |
| TUI 框架   | **标准库 (ANSI)**       | 无需安装任何第三方包，开箱即用                      |
| 打包方式   | 单文件脚本              | 无需安装，直接运行                                 |

## 4. 数据模型设计

### 4.1 FileNode（文件节点）

```python
@dataclass
class FileNode:
    name: str           # 文件名
    path: str           # 完整路径
    size: int           # 文件大小（字节）
    modified: float     # 最后修改时间戳
    extension: str      # 文件扩展名
    parent: DirNode     # 父目录引用
```

### 4.2 DirNode（目录节点）

```python
@dataclass
class DirNode:
    name: str               # 目录名
    path: str               # 完整路径
    size: int               # 目录总大小（字节，含子项）
    file_count: int         # 直接子文件数量
    dir_count: int          # 直接子目录数量
    children: List[Union[FileNode, DirNode]]  # 子节点列表
    parent: Optional[DirNode]                 # 父目录引用
    modified: float         # 最后修改时间戳
```

### 4.3 ScanResult（扫描结果）

```python
@dataclass
class ScanResult:
    root: DirNode                  # 根目录节点
    total_size: int                # 总大小
    total_files: int               # 总文件数
    total_dirs: int                # 总目录数
    scan_duration: float           # 扫描耗时（秒）
    all_files: List[FileNode]      # 所有文件扁平列表（用于排序）
    all_dirs: List[DirNode]        # 所有目录扁平列表（用于排序）
```

## 5. 核心算法

### 5.1 目录扫描算法（后序遍历）

```
function scan_directory(path, parent=None):
    dir_node = new DirNode(path, parent)

    for entry in listdir(path):
        if entry is directory:
            child_dir = scan_directory(entry.path, dir_node)  // 递归
            dir_node.size += child_dir.size
            dir_node.children.append(child_dir)
        elif entry is file:
            file_node = new FileNode(entry.path, entry.size, dir_node)
            dir_node.size += entry.size
            dir_node.children.append(file_node)

    return dir_node
```

**关键点：**
- 使用 **后序遍历**（先递归子目录，再累加大小），确保父目录 size 包含所有后代
- 捕获 `PermissionError`，跳过无权限的目录，不中断扫描
- 忽略符号链接，避免循环引用

### 5.2 排序策略

| 排序模式     | 排序键                       | 默认 |
| ------------ | ---------------------------- | ---- |
| `size-desc`  | `node.size` 降序             | 是   |
| `size-asc`   | `node.size` 升序             | 否   |
| `name`       | `node.name` 字母序           | 否   |
| `type`       | `node.extension` 分组后排序   | 否   |
| `modified`   | `node.modified` 时间倒序     | 否   |

## 7. 命令行接口设计

### 7.1 基本用法

```bash
# 扫描当前目录
python disk_scanner.py

# 扫描指定目录
python disk_scanner.py /home/user/Documents

# 仅显示 Top 20 大文件
python disk_scanner.py /home/user -n 20

# 过滤：仅显示大于 100MB 的文件
python disk_scanner.py /home/user --min-size 100MB

# 过滤：仅显示 .mp4 和 .mkv 文件
python disk_scanner.py /home/user --ext .mp4,.mkv

# 导出为 CSV
python disk_scanner.py /home/user --export result.csv

# 非交互模式：直接输出表格到终端（适合脚本调用）
python disk_scanner.py /home/user --no-interactive
```

### 7.2 参数定义

| 参数               | 类型    | 默认值   | 说明                        |
| ------------------ | ------- | -------- | --------------------------- |
| `path`             | str     | `.`      | 扫描目标路径                 |
| `-n, --top`        | int     | 0 (全部) | 仅显示 Top N 条目            |
| `--min-size`       | str     | `0`      | 最小文件大小过滤             |
| `--ext`            | str     | 无       | 文件扩展名过滤（逗号分隔）    |
| `--sort`           | str     | `size`   | 排序字段: size/name/modified |
| `--order`          | str     | `desc`   | 排序方向: asc/desc           |
| `--export`         | str     | 无       | 导出文件路径（.csv/.json）    |
| `--no-interactive` | flag    | False    | 非交互模式，直接输出          |
| `--no-follow-symlinks` | flag | True   | 不跟踪符号链接               |

## 8. 错误处理策略

| 错误场景               | 处理方式                                     |
| ---------------------- | -------------------------------------------- |
| 路径不存在             | 立即报错退出，提示用户检查路径                 |
| 无权限访问的目录/文件   | 跳过并记录，扫描继续，结束后汇报跳过的数量      |
| 符号链接循环            | 检测并跳过，不陷入死循环                       |
| 磁盘 I/O 错误          | 捕获异常，跳过当前项，继续扫描                 |
| 路径包含特殊字符        | 使用 `os.scandir()` 安全遍历                  |

## 9. 性能设计

### 9.1 性能目标

| 指标           | 目标                              |
| -------------- | --------------------------------- |
| 10万文件扫描   | ≤ 15 秒（SSD）                    |
| 内存占用       | ≤ 200 MB（10万文件节点）           |
| UI 响应        | 排序/切换视图 ≤ 0.5 秒            |

### 9.2 优化策略

- 使用 `os.scandir()` 代替 `os.listdir()`，减少系统调用次数
- 使用 `entry.stat(follow_symlinks=False)` 避免重复 stat 调用
- 扫描过程中显示实时进度条（已扫描文件数 / 当前路径）
- 排序使用 Python 内置 `sorted()`（Timsort，O(n log n)）
- 大数据量时默认只渲染可视区域（rich Table 自带虚拟滚动）

### 13.4 删除功能设计

```
用户选择条目 → 点击「删除选中」
      ↓
显示确认对话框（列出所有待删除项，最多 15 条）
      ↓
用户确认 → 遍历删除（shutil.rmtree / os.remove）
      ↓
显示结果（成功数 + 失败数及原因）
      ↓
自动刷新列表
```

**安全措施：**
- 删除前必须确认，显示完整的待删除列表
- 使用 `shutil.rmtree(ignore_errors=False)` 确保权限错误被捕获
- 删除失败时逐条报告错误原因
- 删除后自动刷新树表，反映最新状态
